Overview

Planning the tasks required to complete an application for managing medical images at align technology.

Problem

It is very difficult to manage so many image documents and recognizing and distinguishing between them in a hospital it can be an X-ray report or someone's test reports and it might get mixed and cause serious problems like it can lead to wrong treatments to the patient and even worse the cause of someone's death because of wrong treatment, so it's a very crucial document and the client came up with an idea to having a Medical Image Manager application so that way they could easily manage the Medical images and also distinguish between them without any issues. so Now They have two developers available to work on the project, and the client would like to have a prototype to review after three sprints. When planning out the tasks required to complete this prototype, they want us to leave space in the last sprint for testing and bug fixes.

Project Reference

The client has provided basic wireframes for what the prototype should look like, the images to populate the application, and a CSV file containing the relevant metadata.
They've also provided the following requirements for the application:

The application should have two main views the user can toggle between:

  • A text view displaying a list of image metadata.
  • An image view displaying the images in a grid.

Clicking an image in the main view should switch to a single view displaying just that image. The image view should have a “back” option to go to the main view. The application should have a control bar supporting the following functions:

  • A field that displays the metadata for the currently selected image.
  • A button to move ahead to the next image. If the main view is open, the next item in the view will be highlighted. If the image view is open, the displayed image will move to the next item in the list.
  • A button to return to the last image. If the main view is open, the previous item in the view will be highlighted. If the image view is open, the displayed image will move to the previous item in the list.
  • A menu to set the magnification of the image.

The application should have a search bar that filters the images displayed in the main view, automatically filtering the view as users enter text into the field.
The application should have a side panel used to navigate through patient names and facilities. Selecting an option will display only images meeting that criteria in the main view. The panel will have two views the user can toggle between:

  • A patient view containing a list of all patients.
  • A facility view containing a list of facilities.

My Role

Front-end Developer responsible for the client's Project Planning.

Solution

We first reviewed the client's requirements for the prototype and then we broke up the project into individual tasks required to be performed in order to complete the application then we scoped out the approximate time we expect each task to take. After that we will create a plan for each sprint and assign each task to a developer by sprint and we will divide the whole project and tasks into three sprints.
